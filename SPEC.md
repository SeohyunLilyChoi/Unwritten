# Unwritten — 기능명세

## 공통 정보

- **서비스명**: Unwritten
- **서비스 성격**: 직장인 암묵지 공유 플랫폼 (직장생활 Q&A + 커뮤니티 + AI 답변)
- **타겟 사용자**: 0~10년차 직장인
- **기술 스택**: React 18, Tailwind CSS 3, Vite
- **브랜드 톤**: 모던 · 심플 · 스마트. 무겁지 않되 신뢰감 있는 분위기
- **컬러**: 화이트/라이트그레이 베이스, 포인트 블루(`#3B5BFF`), `brand-blue` / `brand-blue-light` / `brand-blue-dark` Tailwind 커스텀 토큰으로 등록
- **타이포**: Pretendard 우선, 한글 가독성 중심
- **레이아웃**: 모바일 기준 `max-w-[430px]` 중앙 정렬, iOS safe area(`env(safe-area-inset-*)`) 대응
- **주요 네비게이션**: 하단 탭바 (홈/질문 · 콘텐츠 · 커뮤니티 · 마이페이지)

---

## 화면 1: 홈 (질문 페이지)

**목적**: AI에게 직장 내 고민/암묵지 관련 질문을 하고, 스레드 단위로 대화를 이어가는 메인 화면

### UI 구조

- 상단: `Unwritten` 로고 텍스트 + 우측 "지난 질문" 토글 버튼
- 중간: 질문 스레드 피드 (날짜 구분선 포함, 스크롤)
- 하단 고정: 질문 입력창 (iOS safe area 대응)

### 질문 입력창

- Placeholder: `오늘의 고민은 무엇인가요?`
- `textarea` 멀티라인 입력, `Enter` 전송 / `Shift+Enter` 줄바꿈
- 우측: 전송 버튼 (입력 없을 때 비활성, 회색)
- 히스토리 모드에서 포커스 시 자동으로 현재 세션 모드로 전환
- 콘텐츠 화면에서 "AI에게 질문하기" 탭 시 해당 키워드가 프리필되어 포커스

### 질문 스레드 (`QuestionThread`)

전송 시 AI가 짧은 제목을 생성하고, 해당 제목이 적힌 토글 버튼이 피드에 추가됨.

**토글 버튼**
- 좌측: 파란 점 (제목 생성 중일 때 회색 + pulse 애니메이션, skeleton 바)
- 우측: 접기/펼치기 chevron
- 새로 생성된 스레드는 자동 펼침

**스레드 콘텐츠** (`ThreadContent`)
- 사용자 질문 말풍선 (우측 정렬, 회색 배경)
- AI 답변: 파란 점 레이블 + 헤드라인(굵게) + 본문(핵심 문장 파란 밑줄 강조) + 분석 근거
- 데이터 카드: 제목 + 총 응답자 수 + 막대 바 차트 + `관련 데이터 더보기 >` (첫 번째 메시지에만 표시)
- 관련 커뮤니티 글: 제목 + 본문 2줄 + 댓글/좋아요 수 (첫 번째 메시지에만 표시)
- 답변 피드백: `이 답변이 도움됐나요?` + 👍 / 👎

**꼬리 질문** (스레드 하단 소형 입력창)
- Placeholder: `이 주제로 더 궁금한 게 있나요?`
- 꼬리 질문 말풍선: `bg-brand-blue-light` 배경, 좌측 파란 세로선 구분
- 꼬리 질문 답변: 데이터 카드 · 커뮤니티 글 생략, 피드백 UI 포함
- 마지막 답변 로딩 중일 때 꼬리 질문 입력 비활성

**답변 피드백**
- 👍: 즉시 "피드백 감사해요 👍" 텍스트로 교체
- 👎: 이유 선택 바텀시트 노출 (선택지: "내 상황과 달라요" / "답변이 너무 모호해요" / "정보가 부족해요")

**로딩 순서**
1. 전송 즉시 → 토글 버튼에 skeleton 표시
2. 800ms 후 → AI 생성 제목 표시, skeleton 해제
3. 1,400ms 후 → 답변 렌더링, 스레드 내부 skeleton 해제

### 질문 히스토리 모드

- 헤더 우측 "지난 질문" 버튼으로 `현재 세션 ↔ 히스토리` 전환
- 날짜 구분선과 함께 과거 스레드 목록 표시 (현재 세션도 날짜 구분 적용)
- 히스토리 스레드도 동일한 `QuestionThread` 컴포넌트, 기본 접힘 상태
- 비슷한 질문 감지 시 상단 배너 노출: "이전에 비슷한 질문을 한 적 있어요 → 보러가기"
  - "보러가기" 탭 시 히스토리 모드로 전환
  - X 버튼으로 배너 닫기
  - 감지 기준: 공통 단어 2개 이상

**상태**
- 현재 세션 / 히스토리 모드
- 스레드 접힘 / 펼침
- 제목 생성 중 (skeleton) / 제목 완료
- 답변 로딩 중 (skeleton) / 답변 완료
- 꼬리 질문 입력 전 / 입력 중 / 로딩 중 / 완료
- 피드백 전 / 👍 후 / 👎 이유 선택 중 / 이유 선택 완료
- 비슷한 질문 배너 노출 / 닫힘
- 빈 상태 (첫 질문 전): "첫 질문을 해보세요" 안내

## 화면 2: 콘텐츠 페이지

**목적**: 직장 내 암묵지 관련 콘텐츠를 개인화된 피드 형태로 제공하고, 궁금한 점을 AI에게 바로 이어서 질문할 수 있는 화면

### UI 구조

- 상단: 페이지 타이틀 + 콘텐츠 탭
- 오늘의 추천 배너 (dismissible)
- 키워드 태그 가로 스크롤
- 콘텐츠 피드

### 콘텐츠 탭

`전체` / `투표` / `아티클` / `콘테스트` — 탭 선택 시 해당 유형만 표시

### 오늘의 추천 배너

- 최근 질문 키워드 기반 추천 문구 (예: "출근 시간 고민 중이시죠? 관련 콘텐츠예요")
- 우측 상단 X 버튼으로 닫기 (세션 내 유지)
- 배너 닫힌 후 피드 상단으로 키워드 태그 이동

### 키워드 태그

- 가로 스크롤 (`scrollbarWidth: none`)
- `#전체` 선택 시 필터 해제, 개별 태그 선택 시 해당 키워드 콘텐츠만 표시
- 콘텐츠 탭과 키워드 필터 동시 적용

### 콘텐츠 유형

**① 투표 / 밸런스게임 카드**
- 질문 텍스트 + 선택지 2개
- 선택 전: 기본 버튼 상태
- 선택 후: 결과 막대 바 (선택 항목 파란색, 나머지 회색) + 퍼센트
- 투표 후 직무·연차 필터 탭 노출: `전체` / `내 직무(기획직)` / `내 연차(3년차)`
  - 필터 전환 시 막대 바 width `transition: 0.5s ease` 애니메이션
  - 섹션 타이틀: "기획직 직무는 어떻게 답했나요?" 형태로 동적 표시
  - 해당 필터 데이터 없을 때: "아직 같은 직무의 응답이 없어요" 안내
- 참여 후 `참여완료` 뱃지 + 카드 opacity 0.6 처리
- 하단: "이 내용으로 AI에게 질문하기" 버튼

**② 아티클 카드**
- 썸네일 이미지 영역 + 카테고리 태그 + 제목 + 요약 2줄 + 날짜
- 탭 시 읽음 처리: opacity 0.6 + "읽음" 뱃지 (썸네일 좌상단)
- 하단: "이 내용으로 AI에게 질문하기" 버튼

**콘텐츠 → 홈 연결**
- "이 내용으로 AI에게 질문하기" 버튼 탭 시:
  - 홈 탭으로 이동
  - 입력창에 해당 콘텐츠 맥락 프리필 + 자동 포커스

**상태**
- 추천 배너 노출 / 닫힘
- 투표 전 / 투표 후 전체 결과 / 직무 필터 / 연차 필터 / 필터 데이터 없음
- 콘텐츠 읽기 전 / 읽은 후
- 키워드 필터 전체 / 선택된 태그
- 필터 결과 없음: "해당 키워드 콘텐츠가 아직 없어요"

## 화면 3: 커뮤니티 페이지

**목적**: 사용자들이 직장생활 질문/고민을 올리고 소통하는 공간. AI 묶음 기능과 포인트 동기부여로 참여를 촉진

### UI 구조

- 상단: 타이틀 + 포인트 뱃지 (우측)
- 검색바
- 메인 탭: `전체` / `답할 수 있는 질문`
- `전체` 탭: 트렌드 카드 → 연차·직무 필터 → 정렬 → 게시글 리스트
- 우측 하단 FAB: 글쓰기 버튼

### 포인트 뱃지

- 헤더 우측 고정: 🪙 N P
- 탭 시 포인트 내역 바텀시트 노출 (날짜 / 내용 / 적립 포인트)
- 포인트 0일 때도 뱃지 노출

### 트렌드 카드 (연차·직무 필터 연동)

- 선택된 필터에 따라 타이틀과 키워드 태그 동적 변경
  - `전체`: "커뮤니티 최근 핫토픽"
  - `내 직무`: "{직무}들의 최근 고민"
  - `내 연차`: "{연차}의 최근 고민"
  - `직무+연차`: "{연차} {직무}들의 최근 고민"
- 키워드 태그 탭 시 해당 키워드 게시글 필터링
- 필터 탭 변경 시 선택된 태그 초기화

### 연차·직무 필터 탭

`전체` / `내 직무(기획직)` / `내 연차(3년차)` / `직무+연차` — 네 가지 독립 기준
- 섹션 타이틀 동적 변경: "기획직들의 글", "3년차들의 글" 등
- 필터 결과 없을 때: "아직 같은 직무·연차의 글이 없어요"

### 게시글 카드

**일반 게시글**
- 제목 + 본문 2줄 미리보기 + 작성자(직무/연차) + 댓글 수 / 좋아요 수
- 탭 시 게시글 상세 (`PostDetail`) 이동

**AI 묶음 게시글**
- 좌측 파란 4px 보더 + 연파란 그라디언트 배경으로 시각 구분
- "AI 묶음 N개" 아이콘 뱃지 + "탭해서 요약 보기" 힌트 텍스트
- 탭 시 묶음 상세 (`BundleDetail`) 이동

### 게시글 상세 (`PostDetail`)

- 제목 + 작성자 + 댓글/좋아요 수 + 본문
- 연관 묶음이 있는 경우: "비슷한 질문 묶음 보기 >" 버튼 → `BundleDetail` 이동
- 뒤로가기 버튼으로 리스트 복귀

### AI 묶음 상세 (`BundleDetail`)

- **AI 요약**: 파란 점 레이블 + 헤드라인(굵게) + 근거 본문 + 분석 출처
- **원문 질문 목록**: 각 게시글 accordion 접기/펼치기 (chevron 토글)
- **주목받은 답변**: 채택된 답변 상단 + "채택된 답변" 뱃지, 일반 답변과 배경 구분
- 뒤로가기 버튼으로 리스트 복귀

### 답할 수 있는 질문 탭

- 사용자 직무·연차 기반 매칭 배경 배너 (매칭 기준 안내)
- `AnswerableCard`: "내 경험을 나눠주세요" 레이블 + 매칭 이유 + 질문 내용 + "답변 쓰기" 버튼
- "답변 쓰기" 탭 시 답변 작성 바텀시트 오픈:
  - 원문 질문 미리보기
  - 텍스트 입력창
  - "채택되면 50P 적립돼요 🪙" 안내
  - 등록 버튼 (입력 없을 때 비활성)
- 프로필 미설정 시: "직무와 연차를 설정하면 맞춤 질문을 볼 수 있어요" 빈 상태

**상태**
- 게시글 리스트 / 직무·연차 필터 적용 / 키워드 태그 필터 / 필터 결과 없음
- 일반 상세 (`PostDetail`) / 묶음 상세 (`BundleDetail`) — 리스트 대체 렌더
- `BundleDetail`: 원문 접힘 / 펼침
- `답할 수 있는 질문` 탭: 매칭 질문 있음 / 없음 / 프로필 미설정
- 포인트 바텀시트: 내역 있음 / 없음
- 답변 작성 바텀시트: 입력 전 / 입력 중

## 화면 4: 마이페이지

**목적**: 내 활동 내역 확인 및 계정 관리

### UI 구조

- 상단: 프로필 영역 (직무/연차 + 이메일, 수정 버튼)
- 활동 요약 카드 (포인트 · 쓴 글 수 · 채택 답변 수) — 3열 그리드
- 활동 인사이트 카드
- 메뉴 리스트

### 프로필 영역

- 직무 / 연차 텍스트 (굵게) + 이메일
- "프로필 수정" 버튼 (우측) → 프로필 수정 화면 이동

### 활동 요약

- 보유 포인트 / 내가 쓴 글 수 / 채택된 답변 수 — 3열 카드

### 활동 인사이트 카드

- 활동 기반 인사이트 (예: "당신은 주로 조직문화 관련 질문을 많이 해요")

### 메뉴 항목

- 내가 쓴 글
- 스크랩한 콘텐츠
- 포인트 내역
- 알림 설정
- 앱 정보
- 로그아웃 (위험 액션, 빨간 텍스트)

**상태**
- 로그인 상태 / 비로그인 상태 (로그인 유도 CTA)
- 포인트 0 / 포인트 있음

## 화면 간 연결

| 출발 | 도착 | 트리거 | 전달 데이터 |
|------|------|--------|------------|
| 콘텐츠 → 홈 | AI 질문하기 버튼 | 콘텐츠 제목 기반 프리필 텍스트 |
| 커뮤니티 묶음 카드 → BundleDetail | 탭 | groupId |
| 커뮤니티 일반 카드 → PostDetail | 탭 | post 객체 |
| PostDetail → BundleDetail | "비슷한 묶음 보기" 탭 | relatedBundleId |
| 커뮤니티 포인트 뱃지 → 포인트 바텀시트 | 탭 | — |
| 답할 수 있는 질문 → 답변 작성 바텀시트 | "답변 쓰기" 탭 | post 객체 |
| 홈 비슷한 질문 배너 → 히스토리 모드 | "보러가기" 탭 | — |
